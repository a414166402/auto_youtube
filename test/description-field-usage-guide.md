# Description 字段使用指南

## 功能概述

主体（Subject）的 `description` 字段用于为角色、物品、场景添加简短的描述信息，帮助用户在有多个同类型主体时进行区分和识别。

## 使用场景

### 场景1：创建第一个主体
当创建第一个角色/物品/场景时，description 字段是可选的。

**示例**：
- 名称：主角
- 描述：（可以留空）

### 场景2：创建第二个同类型主体
当创建第2个同类型主体时，系统会显示提示信息，建议填写描述以便区分。

**示例**：
- 名称：角色A
- 描述：紫发女子，穿着白色连衣裙
- 名称：角色B  
- 描述：金发男子，穿着黑色西装

**提示信息**：
```
💡 提示：您正在添加第2个角色，建议填写描述以便区分不同的角色
```

### 场景3：编辑现有主体
可以随时为已有主体添加或修改描述。

## 操作步骤

### 创建主体时添加描述

1. 进入 Settings 页面
2. 选择对应的 Tab（角色库/物品库/场景库）
3. 点击"增加XXX"按钮
4. 填写名称（必填）
5. 填写描述（可选，建议50字符内）
6. 上传参考图（可选）
7. 点击"添加"

### 编辑主体描述

1. 在主体卡片上找到描述区域
2. 点击"编辑"按钮
3. 在输入框中输入或修改描述
4. 按 Enter 键或点击"保存"按钮
5. 按 Escape 键可取消编辑

### 清空主体描述

1. 点击描述区域的"编辑"按钮
2. 清空输入框中的所有文字
3. 按 Enter 键或点击"保存"按钮
4. 描述将被清空，显示为"暂无描述"

## 字段规则

### 字符限制
- 建议长度：50字符以内
- 最大长度：100字符
- 支持所有 UTF-8 字符（中文、英文、数字、emoji等）

### 数据行为
- **不传 description**：保持原有值不变
- **传空字符串**：清空描述（设置为 null）
- **传非空字符串**：更新为新值

### 显示规则
- 有描述：显示描述内容
- 无描述（null）：显示"暂无描述"

## 最佳实践

### 描述内容建议
1. **简洁明了**：用最少的文字描述关键特征
2. **突出差异**：重点描述与其他主体的区别
3. **视觉特征**：描述外观、颜色、服装等可见特征
4. **功能特点**：对于物品和场景，可描述用途或氛围

### 示例

#### 角色描述
- ✅ 好：紫发女子，穿着白色连衣裙
- ✅ 好：金发男子，手持长剑
- ❌ 差：一个人（太模糊）
- ❌ 差：这是游戏中的主角，他有很多技能...（太长）

#### 物品描述
- ✅ 好：金色宝箱，镶嵌红宝石
- ✅ 好：古老的魔法书，封面有神秘符文
- ❌ 差：宝箱（太简单）

#### 场景描述
- ✅ 好：森林场景，阳光透过树叶
- ✅ 好：科幻城市，霓虹灯闪烁
- ❌ 差：一个场景（无意义）

## 技术细节

### API 调用
```typescript
// 创建主体（带描述）
await createSubject(
  'character',
  '角色A',
  '紫发女子，穿着白色连衣裙',
  imageFile
);

// 更新描述
await updateSubject(
  subjectId,
  undefined,  // 不修改名称
  '新的描述内容'
);

// 清空描述
await updateSubject(
  subjectId,
  undefined,
  ''  // 空字符串清空
);
```

### 数据结构
```typescript
interface Subject {
  id: string;
  type: 'character' | 'object' | 'scene';
  name?: string;
  description?: string | null;  // 可为 null
  image_url?: string;
  created_at: string;
  updated_at: string;
}
```

## 常见问题

### Q: description 字段是必填的吗？
A: 不是。description 是可选字段，但建议在有多个同类型主体时填写以便区分。

### Q: 可以使用 emoji 吗？
A: 可以。description 支持所有 UTF-8 字符，包括中文、英文、数字、emoji等。

### Q: 如何删除已有的描述？
A: 点击编辑按钮，清空输入框，然后保存即可。

### Q: description 会影响提示词生成吗？
A: 不会。description 仅用于前端展示和用户识别，不参与提示词生成。

### Q: 第2个主体必须填写描述吗？
A: 不是必须的，但强烈建议填写。系统会显示提示信息提醒用户。

## 总结

description 字段是一个简单但实用的功能，帮助用户更好地管理和识别多个主体。建议在创建第2个及以后的同类型主体时填写描述，以提高工作效率。
